### 2026-05-27 — Day 1 — Basic Syntax: класс-обёртка + имя файла = имя класса

**Pomodoros:** 1/3 (50/10) — заполни сам
**Roadmap progress:** Phase 1 — Basic Syntax — структура файла, класс, точка входа `main`

**Что освоил сегодня (своими словами, без copy-paste):**

- В Java любой код обязан жить внутри класса. Нет верхнего уровня как в JS/Python.
- Внутри класса лежат **методы** (methods) — это то, что в JS зовут "function", когда она живёт внутри класса.
- Для `public class X` файл ОБЯЗАН называться `X.java`. Это правило проверяет компилятор `javac`.
- Причина обёртки в класс: JVM умеет загружать и запускать только классы как единицы. Свободно стоящий код JVM не умеет — у неё нет такого API.
- `javac` и `java` — две разные программы. `javac HelloWorld.java` создаёт `HelloWorld.class` (bytecode). `java HelloWorld` уже просит JVM загрузить и запустить класс.

**Своими словами объясняю Feynman:**

> JVM ищет карандаши по цвету (по имени класса). Если на карандаше написано "розовый", а сам он красный — JVM замечает и указывает на ошибку. Если имя на карандаше совпадает с тем, что ищет — принимает. Так объясняется правило `имя_файла = имя public_class`.
> Дополнительно: код в Java обязан жить внутри класса, потому что у JVM нет команды "выполни вот этот свободный кусок" — только "загрузи класс по имени и запусти его main".

### 2026-05-27 — Day 2 — Basic Syntax: `;` vs `{}` (statement vs block)

**Pomodoros:** 2/3 — заполни сам
**Roadmap progress:** Phase 1 — Basic Syntax — точки с запятой, фигурные скобки, комментарии. Структурная часть Basic Syntax закрыта.

**Что освоил сегодня:**

- `;` — маркер конца **инструкции** (statement). Говорит компилятору: "одна команда закончилась, дальше начинается следующая". В одном методе может быть много инструкций — каждая со своей `;`.
- `{}` — граница **блока** (block). Группирует код в одно целое. Класс — это блок. Метод — это блок. Дальше увижу: `if`, `for`, `while` тоже создают свои блоки.
- Главное озарение: знаки масштабируются по-разному. В методе с 3 инструкциями: 3 `;`, но всё те же 2 пары `{}` (класс + метод). Это и есть доказательство, что они разные структурные категории.
- Комментарии — три формы: `//` (однострочный, как в JS), `/* ... */` (многострочный, как в JS), `/** ... */` (Javadoc — уникален для Java, из него генерируется HTML-документация инструментом `javadoc`).

**Своими словами объясняю Feynman:**

> `;` — это точка в конце предложения. В одном абзаце (метод) много предложений (инструкций), каждое заканчивается точкой. `{}` — это сами абзацы, рамка вокруг группы предложений. Точка не равно рамка: разные структурные знаки на разных уровнях.

### 2026-05-27 — Day 3 — Data Types: int, double, String + lossy conversion

**Pomodoros:** ?/3 — заполни сам
**Roadmap progress:** Phase 1 — Data Types — введение, первые три типа (`int`, `double`, `String`), концепт type-strictness через настоящую ошибку компилятора.

**Что освоил сегодня:**

- В Java каждая переменная имеет ЯВНЫЙ тип, который пишется ПЕРЕД именем. В JS — `let x = 5;` (без типа). В Java — `int x = 5;`.
- Три типа сегодня:
  - `int` — целые числа без дробной части
  - `double` — числа с дробной частью (как `5.5`, `435.21`)
  - `String` — текст (всегда с большой буквы)
- В Java 10+ существует `var`, работает почти как `let` в JS — компилятор сам выбирает тип. Под капотом всё равно конкретный тип, просто короткий путь.
- Java НЕ ПОЗВОЛЯЕТ молча терять данные при конвертации. `int age = 5.5;` → ошибка `incompatible types: possible lossy conversion from double to int`. "Lossy" = с потерей: `.5` отбросилась бы, Java предупреждает.
- Чтобы такая конвертация прошла, надо явно подтвердить — это **type casting**, следующая тема по roadmap.

**Своими словами объясняю Feynman:**

> В JS я мог писать `let age = 5.5;` — JS сам понимает, что это число (единый тип `number` для всех чисел). В Java надо явно указать тип: `int` или `double`. И если я попытаюсь засунуть `5.5` в `int` — Java скажет "стой, ты потеряешь `.5`, подтверди явно". Это защита от случайной потери данных.

**Код, который я написал сегодня:**

- В `HelloWorld.java` объявил `double age = 5.5;` после исправления с `int`.
- В чате (для тренировки) — `int age = 5;`, `String name = "Mike";`, `double money = 435.21;`.

**Ошибки, которые я сделал:**

1. Первая догадка для объявления переменной была `var`. Это реальная фишка Java 10+, но для фундамента надо учить явные типы — `int`/`double`/`String` — а не `var`.
2. Перевод "lossy" как "конвертация теряется" — на самом деле "конвертация **с потерей**". Что-то теряется ВО ВРЕМЯ конвертации, а не сама конвертация исчезает.
3. JS-сторона: сказал "JS выдаёт тип флоат" — в JS тип для всех чисел называется `number` (не `float`), и нет разделения int/float, как в Java.

**Что осталось непонятным:**

- Остальные примитивы Java: `long`, `float`, `byte`, `short`, `boolean`, `char` (следующая сессия)
- Type casting — явная конвертация типов (следующая тема по roadmap)
